4. Implementation Checklist

🔨 Each template TypeScript file must:

  • Accept strongly-typed args interface (not any)
  • Run all user-supplied strings through escapeHtml() before injection
  • Return { subject: string; html: string }
  • Never use external CSS links — all styles must be inline
  • Include the hidden preheader <div> with zero-width non-joiners padding
  • Wrap all layout in role="presentation" tables, never <div> layout

📅 Calendar URL builder (for template 03)

// lib/emails/helpers/calendarUrls.ts export function buildGoogleCalUrl(title: string, start: Date, end: Date, location: string) { const fmt = (d: Date) => d.toISOString().replace(/[-:]/g, '').split('.')[0] + 'Z' const p = new URLSearchParams({ action: 'TEMPLATE', text: title, dates: `${fmt(start)}/${fmt(end)}`, location, }) return `https://calendar.google.com/calendar/r/eventedit?${p}` } export function buildOutlookUrl(title: string, start: Date, end: Date) { const fmt = (d: Date) => d.toISOString() const p = new URLSearchParams({ path: '/calendar/action/compose', rru: 'addevent', subject: title, startdt: fmt(start), enddt: fmt(end), }) return `https://outlook.live.com/calendar/0/deeplink/compose?${p}` } // ICS — serve via /api/demo/ics?id=<demoId> // Set Content-Type: text/calendar; charset=utf-8

5. Email Client Compatibility Notes

Rule Reason
Never use border-radius on <a> tags Outlook ignores it — apply to parent <td> instead
All layout via <table>, not <div> Outlook 2013–2021 uses Word rendering engine, breaks flex/grid
Include role="presentation" on all layout tables Screen-reader accessibility; suppresses table role announcements
Use bgcolor attribute alongside CSS background-color Outlook 2007 ignores inline CSS backgrounds
Gradients degrade gracefully in Outlook Add a fallback background-color (use #7C3AED). MSO conditional comments provided in all headers
Keep width at max-width:600px Gmail clips emails over ~102 KB; 600 px is safe for all clients
Emoji in subject lines Render fine on iOS/Android/Mac Mail. Avoid in Outlook-first B2B contexts (option A subjects are emoji-free)
font-family fallback stack Always: Inter, Arial, Helvetica, sans-serif — Inter loads on Apple/web Gmail, Arial as universal fallback
Star rating links (template 07) Each star is a separate <a> in its own <td> — do NOT use display:flex or background-image stars
Pre-header padding Always add &zwnj;&nbsp; pairs after the preview text to prevent Gmail from pulling body text into the preheader slot
